# Stepbystep

**Stepbystep** - расширение для [Quarto](https://quarto.org/), которое содержит инструменты для организации пошаговых заданий, отслеживания прогресса выполнения упражнений и наглядного оформления инструкций с подсветкой элементов изображений.

## Установка

Добавьте расширение в свой проект Quarto:

```bash
quarto add skyfroger/stepbystep
```

После установки шорткоды и фильтры станут доступны после подключения `stepbystep` к проекту или отдельной странице.

## Локализация

Для локализации элементов интерфейса расширения укажите в YAML атрибут `lang`. По умолчанию используется английский язык. Также доступен русский язык:

```yaml
lang: ru
```

## Шаги задания и самостоятельная работа

Чтобы оформить набор действий в один шаг инструкции, оберните каждое действие в блок с классом `.sbsaction`:

```markdown
:::{.sbsaction}
Первое действие.
:::

Пояснения

:::{.sbsaction}
Второе действие.
:::
```

Задания для самостоятельного выполнения размещаются внутри блока с классом `.sbstask`:

```markdown
:::{.sbstask}
Что нужно сделать самостоятельно.
:::
```

В каждом блоке есть галочка, с помощью которой учащийся может отметить выполненный шаг. Это помогает отследить прогресс выполнения задания.

С помощью шорткода `sbsreset` добавляется кнопка, которая убирает все галочки с выполненных блоков:

```markdown
{{{< sbsreset >}}}
```

## Подсветка элементов изображения

Шорткоды `element` и `pin` создают элементы оформления, которые указывают на подсвечиваемые области изображения:

```markdown
Подключите {{{< element фоторезистор hl="ldr" >}}} одним из контактов к пину {{{< pin 14 hl="pin14-02" >}}}.
```

Параметры:

| Параметр | Описание |
|----------|----------|
| *(первый аргумент)* | Название элемента или номер пина. |
| `hl` | ID элемента для подсветки. Необязательный параметр. |

Изображение, элементы которого нужно подсветить, помещается в блок с классом `.hl-container`. Внутри контейнера размещаются шорткоды `hl`:

```markdown
:::{.hl-container}

![](images/img.png)

{{{< hl pin14-02 pos="39 75.3 3 5" >}}}

{{{< hl ldr pos="56 22.3 18 11" >}}}

:::
```

Параметры шорткода `hl`:

| Параметр | Описание |
|----------|----------|
| *(первый аргумент)* | ID элемента подсветки (должен совпадать с параметром `hl` в шорткоде `element` или `pin`). |
| `pos` | Значения атрибутов `top`, `left`, `width` и `height`, разделённые пробелом. Указываются в процентах. |

> **Важно:** имена идентификаторов в параметре `hl` и в соответствующем шорткоде должны быть уникальными. Иначе стрелки и подсветка будут работать неправильно.

## Хотспот (интерактивные точки)

Шорткод `hs` позволяет добавить активную точку с всплывающей подсказкой на изображение:

```markdown
{{{< hs "Текст подсказки" left top marker="1" >}}}
```

Параметры:

| Параметр | Описание |
|----------|----------|
| *(первый аргумент)* | Текст всплывающей подсказки. |
| `left` | Относительный отступ слева от края изображения (в процентах). |
| `top` | Относительный отступ сверху от края изображения (в процентах). |
| `marker` | Текст внутри маркера. Значение обязательно берётся **в кавычки**. |

Блок с классом `.sbshs` позволяет разместить в хотспоте произвольную разметку, включая блоки кода. Используется внутри `.hl-container`:

````markdown
:::{.hl-container}

![](images/img.png)

{{{< hs "Светодиод с резистором, ограничивающим ток." 73 66 marker="1" >}}}

{{{< hs "Фоторезистор меняет своё сопротивление в зависимости от уровня освещённости" 19.5 55 >}}}

:::{.sbshs left=45 top=20 marker=2}

Следующий код включит светодиод:

```python
from machine import Pin
led = Pin(12, Pin.OUT)
led.on()
```

:::

:::
````

## Постраничный туториал (`pagebypage`)

Блок `pagebypage` позволяет разбить инструкцию для учащихся на набор страниц-слайдов. Каждая страница начинается с заголовка настраиваемого уровня (по умолчанию: третий уровень).

### Настройка

По умолчанию страницы разделяются по заголовкам третьего уровня (`###`). Уровень можно изменить через опцию `headers-level` в YAML-заголовке документа или в `_quarto.yml`:

```yaml
stepbystep:
  headers-level: 4
```

### Разметка

Оберните всё содержимое туториала в блок с классом `.pagebypage`. Каждая страница должна начинаться с заголовка заданного уровня:

```markdown
:::{.pagebypage}

### Первая страница

Содержимое первой страницы...

:::{.sbsaction}
Первое действие.
:::

### Вторая страница

Содержимое второй страницы...

:::{.sbstask}
Задание для самостоятельной работы.
:::

:::
```

### Навигация

Туториал предоставляет меню навигации с нумерованными индикаторами страниц и кнопками «Назад»/«Вперёд». Пройденные страницы помечаются визуально.

### Совместимость с другими элементами

Внутри `.pagebypage` можно использовать любые другие элементы stepbystep: `.sbsaction`, `.sbstask`, `.hl-container`, `element`, `pin`, `hs` и `sbshs`.

## Советы по разметке

1. **Уникальность идентификаторов.** Имена в параметре `hl` шорткодов `element`, `pin` и `hl` должны быть уникальными.
2. **Координаты в процентах.** Параметр `pos` в шорткоде `hl` и атрибуты `left`/`top` в `hs`/`sbshs` задаются в процентах относительно размеров изображения.
3. **Кавычки для marker.** Значение параметра `marker` в шорткоде `hs` обязательно берётся в кавычки, даже если это число.
4. **Вложенность контейнеров.** Для `sbshs` с кодом используйте 4 двоеточия (`::::`) для внешнего блока, чтобы избежать конфликтов с внутренними блоками из 3 двоеточий (`:::`).
5. **Фильтр в YAML.** Не забудьте добавить `filters: - stepbystep` в заголовок документа.
